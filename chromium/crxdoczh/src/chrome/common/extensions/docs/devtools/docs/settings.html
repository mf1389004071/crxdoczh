设置

要修改开发者工具设置：

  • 单击设置图标 打开 General（常用）设置窗格进行更改。您也可以使用 ? 快捷键打开设置窗格。

General（常用）

Disable cache（禁用缓存）

只有当开发者工具打开时避免资源缓存，如果开发者工具关闭了就不会禁用缓存。

Disable JavaScript（禁用 JavaScript）

选中该选项后立即暂停对应标签页中所有 JavaScript 的运行。

注意：禁用缓存和禁用 JavaScript 只会在开发者工具打开时适用，启用时适用于打开了开发者工具的网页。

Enable Ctrl + 1-9 shortcuts to switch panels（启用快捷键 Ctrl + 1～9 切换面板）

您可以使用 Ctrl + 1-9 快捷键跳转到 Chrome 浏览器打开的多个标签页中的某个标签页，如果标签页数目超过 9 个，1～8 对应前 8 个标签页，9 对应最后一个标签页。该设置可以使开发者工具也具有这项功能，这样您可以快速在面板间切换。

注：启用该设置后可能会和其他应用程序中使用的快捷键冲突。

Appearance（外观）

Split panels vertically when docked to right（在右侧固定时垂直分隔面板）

启用该设置后会改变面板布局，使面板的主要部分放置在侧边栏顶部。如果在小屏幕上它们并排显示的话可能没有足够的水平空间，这时该选项就很有用。

Elements（元素）

Color format（颜色格式）

  • As authored（使用编写的格式）（样式表中编写的代码）
  • HEX：#DAC0DE
  • RGB：rgb(128, 255, 255)
  • HSL：hsl(300, 80%, 90%)

颜色格式设置允许您控制元素面板中的样式侧边栏内如何显示颜色代码。除了控制颜色格式的设置选项外，您还可以单击样式侧边栏顶部的齿轮按钮更改颜色代码的格式。

选择 As authored（以编写的格式显示）会使用样式表定义的属性中的颜色格式。

Show user agent styles（显示用户代理样式）

您可能会发现在元素面板的样式侧边栏中显示用户代理样式会很有用。

用户代理指浏览器，每个浏览器都会实现默认的样式表，包含基本的样式规则，应用于网页中的 DOM 元素。例如，如果您试着删除两个元素之间的空白时遇到困难，可能是因为用户代理样式表为这种类型的元素添加了默认外边距或内边距。

Word wrap（自动换行）

和任何一种文本编辑器一样，您可以选择为元素面板中的代码自动换行。

Show Shadow DOM（显示 Shadow DOM）

通过 Shadow DOM，元素可以有一种新的节点与之关联，这种新节点称为 shadow root。拥有与之关联的 shadow root 的元素称为 shadow host。shadow host 的子节点不会渲染，而 shadow root 的内容却会渲染。

  • Shadow DOM 201
  • Shadow DOM 301
  • Shadow DOM Polyfill
  • Visualizing Shadow DOM Concepts（以图形方式展现 Shadow DOM 概念）
  • Define presentational boxes using shadow DOM（使用 Shadow DOM 定义展示框）

Show rulers（显示标尺）

该选项会在视图顶部、左侧和底部显示标尺覆盖层。

Sources（源代码）

Search in content scripts（在内容脚本中搜索）

内容脚本是 Chrome 扩展程序中包含的 JavaScript 文件，在网页的上下文中，但是又在与原始网页的 JavaScript 完全隔离的作用域内运行。这样一来，内容脚本和网页中的脚本就不能通过普通的方法互相交互。

在源代码面板中查看 Content Scripts（内容脚本）标签页时，您可以看到扩展程序（或 User Script，在 Chrome 浏览器中编译为扩展程序）添加的脚本，还有浏览器内建的内容脚本，特别是扩展程序可以使用的 API。

注：开发 Chrome 应用或扩展程序时启用该设置，以便在这些原生 API 脚本中搜索。其他情况下启用该设置没有什么用处。

Enable JS source maps（启用 JS 源代码映射）

如果您的代码经过连接和压缩，在调试时就很难知道某一段代码在哪个文件中。启用该设置在调试 JavaScript 并且使用源代码映射时会很有用。

Enable CSS source maps（启用 CSS 源代码映射）

为预处理器（例如 Sass）生成的 CSS 文件启用源代码映射。

有关详情请参见与 CSS 预处理器协作。

Auto-reload generated CSS（自动重新加载生成的 CSS）

只有在启用 CSS 源代码映射打开时才能使用，决定在源代码保存后是否重新加载生成的 CSS 文件。

Detect Indentation（检测缩进）

指定开发者工具中编辑代码时如何缩进。

  • 2 个空格
  • 4 个空格
  • 8 个空格
  • Tab 制表符

Show whitespace characters（显示空白字符）

该选项会在源代码面板中为空格和制表符显示点。

Profiler（性能分析器）

High resolution CPU profiling（高精度 CPU 性能分析）

使您在火焰图中可以将图象放大至十分之一毫秒来查看。

Console（控制台）

Log XMLHttpRequests（记录 XMLHttpRequest）

在控制台中显示 XHR 请求对象，展开后可以显示请求详情。

Preserve log upon navigation（导航时保留日志）

在网站的多个网页之间导航时，您可以选择不要在每次网页加载时清除控制台日志，这样您可以查看不同网页之间的输出历史记录。

注：这两项设置都可以右键单击控制台更改。

Extensions（扩展程序）

Open links in: a panel chosen automatically（在自动选择的面板中打开链接）

Workspace（工作空间）

工作空间允许您选择文件系统中的自定义目录，使您可以随时在源代码面板中编辑。它可以是某个项目的目录，也可以是包含多个不同项目的目录。

要使用该功能，请打开设置窗格中的 Workspaces（工作空间）标签页。您可以看到Add folder（添加文件夹）按钮，允许您添加需要编辑的本地目录（例如项目根目录）。

一旦您添加文件夹后，您就可以随时在源代码面板中查看、编辑和保存文件夹中的文件，文件的所有更改都会保存到目录中包含的本地文件中。

除了在工作空间中添加文件系统外，您还可以将文件 URL 映射至本地计算机上的文件路径。
