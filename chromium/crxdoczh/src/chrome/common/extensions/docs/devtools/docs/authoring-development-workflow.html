编写和开发的工作流程

开发者的工作流程通常涉及到实现某个目标的一步步操作。使用开发者工具编写代码可以优化这样的工作流程，节约常见任务的时间，例如寻找文件或函数、保存脚本或样式表的更改、保存常用代码片段，或者重新安排布局，以便更适合您的需要。

在这一部分中，我们将探索一些技巧，使您更高效地在开发者工具中工作。

在右侧固定，用于垂直分割编辑

您可能觉得开发者工具固定在窗口底部提供了很大的横向空间，但是纵向空间太小了。在右侧固定可以使开发者工具在窗口右侧显示，并且仍然附加在窗口中，这样您就可以审查左边的当前网页，同时在右边调试。

在下面这几种情况下这种功能非常有用：

  • 您可能使用的是宽屏显示器，希望尽可能增加可用空间，以便审查和调试您的代码
  • 您可以改变分割线的位置，使网页小于 400 像素（目前 Chrome 窗口的最小宽度），以便测试调整大小后的布局
  • 垂直空间下更容易调试较长的脚本

导航到您要调试的 URL，然后单击开发者工具右上角的布局图标 并长按鼠标，切换在右侧固定与固定在窗口底部这两种模式。

注：开发者工具会记住您上一次的选择，所以您可以在这两种常用选项间来回切换。

这样操作后会展开菜单，显示可用的布局选项。一旦您选择了某个选项，布局就会立即更改，体现您的更改。

注：开发者工具在每个标签页中可以有各自的自定义布局，也就是说可以让一个标签页的开发者工具固定在右侧，而另一个固定在窗口底部。

搜索、导航和过滤

根据文件名过滤脚本、样式表或代码片段

对开发者的工作流程来说，快速定位文件的能力是很重要的。开发者工具允许您在所有脚本、样式表和代码片段文件中搜索，您可以使用以下快捷键：

  • Ctrl + O (Windows、Linux)
  • Cmd + O (Mac OSX)

无论您目前在哪个面板都能使用这些快捷键。对于这一个待办事项应用来说，使用其中某个快捷键后会进入 Sources（源代码）面板，并显示一个搜索框，列出所有可以审查的文件。

我们可以从这里过滤出具体的文件（例如文件名中包含“script”）或者选择文件进行查看或编辑。

注：我们在所有对话框中都支持首字母匹配，例如要打开 FooBarScript.js，您可以只输入 FBaSc，这样可以节约时间。

在当前文件中搜索文本

您可以使用以下快捷键在当前文件中搜索某个字符串：

  • Ctrl + F (Windows、Linux)
  • Cmd + F (Mac OSX)

在搜索框中输入关键词后，按下回车键移动到第一个匹配结果，继续按回车键会显示之后的结果。您也可以像下面显示的那样使用 ↑ 和 ↓ 箭头键。

在当前文件中替换文本

除了支持在当前文件中查找文本外，开发者工具还可以将个别或所有出现的文本替换为新的值，选中“Replace”（替换）后就会显示第二个输入框，允许您指定替换文本。

在所有文件中搜索文本

如果您希望在网页加载的所有文件中搜索某个字符串，您可以使用以下快捷键打开搜索窗格：

  • Ctrl + Shift + F (Windows、Linux)
  • Cmd + Opt + F (Max OSX)

这种方式支持正则表达式以及区分大小写的搜索。

使用正则表达式搜索

如果要使用正则表达式搜索，只要在 Search sources（搜索源代码）框中输入表达式，选中“Regular expression”（正则表达式）并按回车键。

在上图的例子中我们可以看到如何查找所有匹配 <div></div> 标签的内容。

过滤文件中的函数或选择器

如果您需要更精细的粒度，还可以导航到（或搜索）文件中的某个 JavaScript 函数或 CSS 规则。

导航到您选择的某个网页，然后进入 Sources（源代码）面板。您可以使用以下快捷键打开函数/选择器搜索框：

  • Ctrl + Shift + O (Windows、Linux)
  • Cmd + Shift + O (Mac OSX)

取决于选择的文件类型，您会看到所有可用的 JavaScript 函数或 CSS 声明。开始键入函数或声明的名称，就能过滤结果列表，也可以直接选择某个结果，就能进入当前文件中定义它的位置。

跳转到行号

开发者工具还支持跳转到编辑器中的某个行号。要打开行号对话框，只要选择要编辑的文件，并使用以下某个快捷键显示行号选择器：

  • Ctrl + L (Windows)
  • Cmd + L (Mac OSX)
  • Ctrl + G (Linux)

实时编辑脚本和样式

开发者工具支持样式和脚本的实时编辑，而不用刷新整个网页。这样可以帮您测试设计更改、使用 JavaScript 函数原型或代码片段。

脚本更改只会在求值的时候执行，也就是说网页加载后不会运行的代码修改后也没有效果。之后的某个阶段执行的代码，例如 mouseover 处理程序或单击事件的回调函数，就可以实时更改和测试。

样式

编辑样式的工作流程也是类似的。打开开发者工具并切换到 Elements（元素）面板，在右侧有一些子面板，包括 Styles（样式）。审查网页中的元素就会在样式中显示应用于当前节点的属性列表，按照选择器分组。

“element.style”部分显示网页标签中 style 属性设置的属性。

接下来一部分是“Matched CSS Rules”（匹配的 CSS 规则），显示匹配选定节点的选择器、它们的属性和值以及规则来源的文件名和行号。匹配节点的选择器用黑色显示，其他选择器则为灰色。这样非常方便，使您看的时候更容易区分选择器。

在某个子面板中更改任意 CSS 属性，例如元素边框或大小，更改就会立即应用，并显示在浏览器主窗口中。

回到“Matched CSS Rules”（匹配的 CSS 规则）面板，单击规则旁的样式表链接，就能带您进入 Sources（源代码）面板。这样可以显示完整的样式表，并定位到相关 CSS 规则所在的行号。

在这里，您可以和普通的文本编辑器一样修改文件，而且所作更改会实时显示。

另存为

一旦您对文件的更改满意之后，就可以保存了。

要做到这一点，首先确保您在文本编辑器视图中，打开了您修改的文件。您可以手动在左侧边栏中的 Sources（源代码）标签页中定位文件：

或者在 Elements（元素）-> Styles（样式）面板（对于 SASS/CSS）中单击文件名（例如 styles.css）：

接着，右键单击文件名或文本编辑器内部，并单击“Save As”（另存为）。这一操作会打开一个对话框，允许您覆盖已有文件。

之后进行保存（使用同一菜单中的 Save（保存）或 Ctrl/Cmd + S 快捷键）会保存到同样的位置。

本地更改

开发者工具还维护了本地文件所有修改的版本历史记录。如果您曾经编辑过脚本或样式表，并使用开发者工具保存更改，您可以右键单击源代码中的文件名（或者在源代码区域内），并选择“Local modifications”（本地更改）查看历史记录。

Local modifications（本地更改）面板就会出现，并显示：

  • 更改的 diff 差异
  • 作出更改的时间
  • 更改的文件所在的域名

此外还有一些链接。revert（恢复）链接会恢复文件的所有更改，回到原始状态，并删除版本历史记录。

Apply original content（应用原始内容）操作的效果是一样的，但是会保留版本历史记录，以防您需要回到某个变更集。

最后，apply revision content （应用版本内容）会应用某个时间特定版本中的更改。

自定义 JavaScript 代码片段

有时候您希望能保存较小的脚本、小书签和实用程序，这样当您在浏览器中调试的时候随时都可以使用它们。代码片段是开发者工具中的新功能，使这种工作方式成为了可能，允许您在 Sources（源代码）标签页中创建、保存和运行 JavaScript。

代码片段在一些情况下非常有用：

  • 小书签——所有小书签都能以代码片段的方式保存，特别是您需要编辑的那些。
  • 实用程序——用来和当前网页交互的调试帮助程序可以保存并调试，您可以参见社区创建的这类实用程序列表。
  • 调试——代码片段提供了一种多行的控制台，具有语法高亮和保存功能，调试超过一行的代码时非常方便。
  • 动态修补代码您需要在运行时修补的代码可以通过代码片段完成，尽管很多时候您可以在 Sources（源代码）标签页中实时编辑代码。
Brian Grinstead 为开发者维护了一个有用代码片段的仓库，可以通过 bgrins.github.io/devtools-snippets 访问。

入门

要想开始使用代码片段，请导航到 Sources（源代码）标签页。如果您没有在该标签页中进行过其他工作，您会看到如下所示的默认布局。

单击左上角的布局切换按钮，显示其他面板。您会看到 Sources（源代码）、Content scripts（内容脚本）和一个新标签页 Snippets（代码片段）。单击它进入代码片段。

创建代码片段

代码片段通过两个面板工作。左边的面板（与 Sources（源代码）类似）是文件列表，选择代码片段文件后就会在右边的编辑器中打开。这种体验与 Sources（源代码）标签页中选择脚本或样式表进行编辑类似。

右键单击文件列表内部，并选择“New”（新建），就能创建新的代码片段文件。

代码片段文件名

代码片段的文件名可以自动生成，但是代码片段创建后您也可以自定义文件名。

在此之后，如果您想重命名文件，只要在文件列表中右键单击它并选择“Rename”（重命名）。如果需要的话您还可以删除（“Remove”）代码片段。

编辑和运行代码片段

从文件列表中选择某个代码片段文件，右边内建的编辑器中就会打开它。在这里您可以编写或粘贴任何 JavaScript 代码（即您的代码片段），包括函数和表达式。

如果文件名前有 *，它表示代码片段已经修改，还没有保存。

如果要运行代码片段，右键单击文件列表中的文件名，并选择“Run”（运行）。您也可以单击 Run（运行）按钮。

如果代码片段有控制台输出，它们会显示在编辑器下方的控制台中。

注：也可以使用快捷键方便地运行代码片段——只要选择您的代码片段，并使用 Ctrl/Cmd + Enter 就能运行。

如果您需要的是在控制台中执行代码片段中的某几行，您只要在编辑器中选中这几行，右键单击选定内容并选择“Evaluate in Console”（在控制台中求值）。对应的键盘快捷键是：Ctrl + Shift + E。

与运行类似，表达式的输出可以在编辑器下方的控制台中查看。

本地更改

与源代码标签页类似，代码片段也支持查看文件的本地更改，还可以恢复到版本历史记录中的某一次编辑。

在本地保存更改后，右键单击编辑器窗口并选择“Local modifications...”（本地更改...），就能访问这一项功能。

断点、监视表达式及其他

您可以在源代码面板中使用的其他特性，例如添加监视表达式、断点、作用域内的变量以及保存文件，在代码片段中也能使用。

阅读源代码面板的相关文档，进一步了解如何使用这些特性。

保存代码片段

代码片段可以保存，以便今后通过开发者工具代码片段标签页访问，也可以导出至系统中的某个新文件。要显示代码片段的保存选项，右键单击文本编辑器内部，显示编辑器菜单。

Save（保存）将更改保存到现有的代码片段文件，而 Save As（另存为）允许您将代码片段保存到选定位置的新文件中。

注：代码片段保存在开发者工具的 localStorage 中。进行保存/另存为操作时，您可以将代码片段绑定到某个文件，就像其他脚本一样保存。

代码片段的导航

与源代码中的脚本和样式表类似，代码片段也支持特定代码片段文件、函数或行号的导航，只要使用我们前面提到的快捷键即可。
