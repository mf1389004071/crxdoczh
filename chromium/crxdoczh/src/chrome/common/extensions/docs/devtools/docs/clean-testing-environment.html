创建干净的测试环境

干净的测试环境使您可以很容易确定网站或网上应用的失败点，而不用考虑是否是后台的其他程序导致速度缓慢或失败。

应用、扩展程序、后台进程和不必要的标签页都会影响您的性能分析结果，应该尽可能避免，在您使用时间线、性能分析 或 about:tracing 进行精确的性能评测时这一点尤其重要。

要想设置干净的测试环境：
  • 确保 Chrome 浏览器以干净的配置文件运行：

    • 方法 1：使用隐身模式。打开Chrome 菜单 并选择打开新的隐身窗口，或者使用 --incognito 参数启动 Chrome 浏览器。

    • 方法 2：使用干净的 Chrome 浏览器配置文件，单击 Chrome 窗口右上角的用户图标，并选择添加人员。

       

      要确保干净的配置文件不会继承任何扩展程序或设置，请使用 --user-data-dir 参数指定 '/dev/null' 路径启动 Chrome 浏览器。例如：

      • OSX: open -a "Google Chrome" --args --user-data-dir=/dev/null
      • Linux: google-chrome --user-data-dir=/dev/null
      • Windows: "C:\Users\username\AppData\Local\Google\Chrome\Application\chrome.exe" --user-data-dir=/dev/null

      这样可以确保您使用完全为空的配置文件，而不是以只读方式加载用户的配置文件。


    • 关闭其他标签页以及您在后台运行的进程/应用程序。如果您需要在后台运行这些应用，请使用系统监视器确保应用不会占用太多 CPU 或内存。
    • 关闭您的系统中可能启用的动态 CPU 超频。
    • 确保您使用最新编译的 Chrome 浏览器。您可以使用 http://download-chromium.appspot.com 获取最新编译的版本。
    • 如果您通常使用 Chrome Canary 作为开发浏览器，遇到问题后应该在 Chrome Stable 中也测试一下，确保问题与新特性或浏览器漏洞无关。
